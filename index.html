Why oh why?

  1. At the very least you should be capable

    even if you don't intend to use it every day

    This editor is everywhere

  3. Especially for servers

    Who lives their life never having to modify a config file?

Things to know

  1. Modal Interface (Input vs. Normal mode)

    This is different from most other editors you know

  2. Editing (In input mode)

    You should already have these skills

  3. Saving/Quitting (In normal mode)

    :wq

Modal Interface

Normal Mode

  1. Command Your Vim

    This is where you do meta operations on your document/vim environment

  2. Dragons live here

    A few keystrokes can drastically alter your stuff

    i.e., your input may destroy your document. Don't touch anything.

  3. We'll get to what you can do in normal mode later.

Input Mode (IS CAKE)

  1. i to enter Input mode

    CASE SENSITIVE

  2. Arrow keys to move around, type type to write stuff

  3. ESC to leave

Normal Mode - Cont

  1. ESC to return here

    Yes, I intentionally repeat this

  2. :wq to Write+Quit

    :w will Write the file, :q will Quit

    :q! will force Quit

ESC is your buddy

Now try it

Vim a file, enter input mode, type in some stuff, leave input mode, quit

  1. $vim test

  2. Notes:

    • i for input mode
    • ESC to leave input mode
    • :wq to write+quit
